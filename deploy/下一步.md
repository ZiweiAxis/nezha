# 下一步（当前推荐）

**当前状态**：天枢默认连 NAS Synapse（xyin.oicp.net），secret 已与紫微一致；原生渠道演示支持自动邀请 @hulk:xyin.oicp.net。

---

## 1. 端到端验证：原生渠道（推荐先做）

在本机启动天枢 + 谛听（不配飞书也可），确认手机 Element（hulk）能收到邀请并看到消息。

```bash
# 仓库根目录
cp deploy/env.example .env   # 已有可跳过；可选填 FEISHU_*、保留 REGISTRATION_SHARED_SECRET
cp deploy/docker-compose.integration.yml docker-compose.integration.yml
podman-compose -f docker-compose.integration.yml --env-file .env up -d

# 等天枢就绪（约 20s）后
curl -s http://localhost:8082/health
curl -s http://localhost:8082/api/v1/delivery/native-demo
```

在手机 Element（@hulk:xyin.oicp.net）接受房间邀请，应看到「【原生渠道验证】审批请求」消息。  
详见 **`deploy/原生渠道验证步骤.md`**。

---

## 2. 飞书端到端（需飞书应用）

若要跑「Agent 注册 → 飞书确认 → 下指令 → 谛听拦截 → 飞书审批」完整流程：

- 在 `.env` 中填写 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`，并按 **`deploy/飞书对接配置说明.md`** 配置权限与回调。
- 启动后按 **`docs/open/technical/核心目标-飞书端到端流程.md`** 操作。

---

## 3. 可选：审批回执（原生渠道）

当前 native-demo 只验证「消息投递到 Matrix 房间」。若要做「用户回复/点击链接完成审批」：

- **方式一**：房间内回复「通过」/「拒绝」→ 天枢 sync 解析并回传谛听（若已实现）。
- **方式二**：消息内带审批链接（指向天枢 HTTP 回调）→ 天枢实现回调并在 payload 中写链接。

见 **`deploy/原生渠道验证步骤.md`** §4。
