## 1. 系统概述

紫微 (Ziwei) 是一个企业级 AI 智能体治理基础设施，旨在为智能体提供安全、可控、可审计的运行平台。平台采用分层解耦、开放接入的架构设计。

### 1.1 核心组件一览

| 组件 | 中文名 | 功能定位 | 技术栈 | 状态 |
|------|--------|----------|--------|------|
| **天枢 (Tianshu)** | 消息枢纽 | 飞书 ↔ Matrix 双向消息桥接、Agent 注册与身份管理 | Python | 开发中 |
| **獬豸 (Xiezhi)** | 策略引擎 | 策略管理、风险评估、人工审批、审计存证 (PDP+PAP) | Go | MVP |
| **悟空 (Wukong)** | 执行控制器 | Agent 端策略执行、任务挂起、审计上报 (PEP) | TypeScript | 开发中 |
| **太白 (Taibai)** | Python SDK | 多语言 SDK、适配器框架、协议定义 | Python | MVP |

## 2. 悟空 (Wukong) 重新定位

### 2.1 为什么叫"执行控制器"而不是"拦截器"

**当前问题**：
- "拦截器" (Interceptor) 通常指网络层面的请求拦截
- 悟空拦截的不仅是网络请求，还包括：
  - 本地文件操作 (file write/read)
  - 命令执行 (exec:run, exec:sudo)
  - API 调用
  - Tool 调用

**更准确的定位**：
- **执行控制器** (Enforcement Controller / PEP)
- 负责在 Agent 执行操作前进行策略检查
- 根据策略决定：放行、拒绝、还是转审批

### 2.2 核心职责

| 职责 | 说明 |
|------|------|
| **策略接收** | 接收并存储獬豸下发的策略规则 |
| **本地评估** | 在 Agent 端进行快速的策略评估 |
| **执行控制** | 根据评估结果决定放行/拒绝/转发 |
| **任务挂起** | 审批期间挂起任务，审批通过后恢复执行 |
| **审计上报** | 将操作日志上报给獬豸 |

### 2.3 与"网关"的区别

| 特性 | 网络网关 | 悟空 (执行控制器) |
|------|----------|------------------|
| 拦截对象 | 网络请求 (HTTP/gRPC) | 所有操作 (文件/Exec/网络/Tool) |
| 部署位置 | 独立服务，流量经过 | 嵌入 Agent 进程内部 |
| 决策方式 | 中心化决策 | 本地快速决策 + 中心审批 |
| 延迟影响 | 有网络延迟 | 无额外网络延迟 |

### 4.2 獬豸 (Xiezhi)

**位置**: `~/workspace/ziwei/xiezhi/`

**功能**:
- 零信任身份验证 (L0)
- 策略引擎与风险评估 (L1-L2)
- 人机协同审批 (L3)

| L4 | 审计追溯 | `audit/store.go` | JSONL/PostgreSQL/ClickHouse 存储 |

## 7. 关键技术

### 7.1 Matrix 通信

**Matrix 服务器**: `xyin.oicp.net:8008`

**用户身份 (DID)**:
| 身份 | 注册方式 | DID 示例 |
|------|----------|----------|
| Owner (真实人类) | 自己注册 | `did:human:xxx` |
| 审批人 | 固定配置 (hulk) | - |
| 獬豸 | 系统初始化自动注册 | `did:agent:xiezhi` |
| Agent | 悟空触发自动注册 | `did:agent:xxx` |

**紫微 Matrix 扩展事件**:
- `m.agent.register`: 智能体注册请求
- `m.agent.identity`: 身份下发与更新
- `m.agent.approval`: 审批事件
- `m.agent.action`: 操作上报
- `m.agent.audit`: 审计存证回执

### 7.2 CHEQ 确认引擎

CHEQ (Confirmation Engine) 是獬豸的人机协同核心，负责创建和管理审批确认对象。

**ConfirmationObject 状态**:
- `pending`: 等待审批
- `approved`: 已批准
- `rejected`: 已拒绝
- `expired`: 已过期

**审批策略**:
- `any`: 任一审批人批准即可
- `all`: 所有审批人都必须批准

### 7.3 WebSocket 广播

獬豸支持 WebSocket 长连接，用于实时审批事件推送。

**相关配置**:
```yaml
delivery:
  feishu:
    use_long_connection: true
    poll_interval_seconds: 2
```
