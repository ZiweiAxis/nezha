# 紫微平台层职责梳理

**目的**：明确 ziwei 平台层（天枢、太白、獬豸）的职责边界，避免混淆。权威表述以各子项目（獬豸等）自身文档为准，本文为根侧统一口径。

---

## 1. 三层职责（一句话）

| 层 | 职责 | 不做什么 |
|----|------|----------|
| **天枢** | 通信枢纽与身份控制平面；联络 Agent（或经太白）收发指令与结果。 | 不执行业务逻辑；不实现监听机制。 |
| **太白** | 联络 Agent 与天枢：接收天枢信息、转发 Agent；Agent 按天枢规范输出、回传天枢。 | 不执行业务逻辑；不实现监听机制；不处理策略/审批。 |
| **獬豸** | **一套机制，达到监听的效果**。提供策略、鉴权（如 /auth/exec）、代理、CHEQ、审计、存证等机制，使智能体的执行与工具调用**可被监听**（可拦截、可审批、可审计）。 | **不执行**业务逻辑；执行在 Agent 侧。獬豸是机制，不是「执行者」也不是「监听者」角色本身——通过这套机制**达到**监听效果。 |

---

## 2. 执行归属

- **执行**（脚本、命令、工具调用、业务逻辑）：在 **Agent** 侧。
- 天枢、太白、獬豸均**不执行**用户/智能体发起的业务操作。獬豸在「执行」链路中的角色是：当 Agent 要执行或发起请求时，**经獬豸的机制**（如 /auth/exec、代理流量经獬豸）实现**监听**（鉴权、审批、审计），从而**达到**监听效果。

---

## 3. 獬豸：机制与监听效果

- 獬豸文档中已说明：獬豸是一套**机制**，**达到监听的效果**。
- 即：通过策略引擎、执行层鉴权（/auth/exec）、HTTP 代理、CHEQ 审批、Trace、链上存证等**机制**，使智能体行为可被观测、可拦截、可审批、可存证——即**监听效果**。獬豸不替代 Agent 执行，也不以「负责监控」的身份出现，而是**提供机制**，由这些机制共同**达到**监听效果。

---
