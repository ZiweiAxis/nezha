# 紫微文档中心

紫微主仓 **仅做本地版本管理**，文档按**开放/内部**划分目录，便于分类；所有目录均可纳入主仓本地版本管理。

---

## 目录划分（按公司运营视角）

| 目录 | 用途 | 说明 |
|------|------|------|
| **open/** | 对外、可开源 | 技术、产品、社区、治理（脱敏） |
| **business/** | 商业类 | 商业策划、BP、战略、运营策划 |
| **ip/** | 知识产权类 | 专利、软著、版权、商标 |
| **internal/** | 内部通用 | 流程、制度、会议纪要等（主仓仅本地管理时可一并纳入） |

---

## 各目录说明

### open/ — 可开源文档

- **open/technical/** — 技术方案、架构、开发指南、跨子项目技术约定  
- **open/product/** — 产品介绍、路线图（脱敏）、对外白皮书  
- **open/community/** — 社区指南、贡献说明、开源治理  
- **open/governance/** — 项目/开源治理（可对外部分）

子仓库中「不适合放入子仓」的**非敏感**通用文档，可迁移至紫微 `docs/open/` 下对应子目录。

### business/ — 商业类（内部，不提交）

- 商业计划书、融资材料  
- 战略规划、市场/运营策划  
- 竞品分析、定价策略等  

子仓库内涉及敏感的商业策划类文档，统一放入此处，**不纳入 git**（主仓 `.gitignore` 已忽略）。

### ip/ — 知识产权类（内部，不提交）

- **ip/patent/** — 专利交底、申请材料  
- **ip/copyright/** — 软著、版权登记材料  
- **ip/trademark/** — 商标相关  

### internal/ — 内部通用（不提交）

- 内部流程、制度、审批模板  
- 会议纪要、决策记录  
- 人事/组织相关（若需集中存放）

---

## 子项目文档迁移约定

- 子仓库（天枢、獬豸）中**商业策划、BP、敏感内部信息** → 迁移到紫微 `docs/business/` 或 `docs/internal/`，子仓内删除或改为占位说明「见紫微 docs/business/…」。  
- 子仓库中**知识产权相关**（软著、专利等） → 迁移到紫微 `docs/ip/`。  
- 子仓库中**可对外的技术/产品/社区文档** → 可保留在子仓，或复制/链接到紫微 `docs/open/` 以做统一入口（按需选择）。

主仓仅做本地版本管理时，所有 docs 目录（含 business/ip/internal）均会被主仓 git 跟踪，无需对文档目录做 ignore。
