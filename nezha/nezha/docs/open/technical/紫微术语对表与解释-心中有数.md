# 紫微术语对表与解释（心中有数）

**用途**：一份可记住的术语对照与解释，便于对外沟通、竞品对比、生态对标时心中有数。  
**版本**：v1.0 | **日期**：2026-02-15

---

## 一、紫微内部术语（子项目与产品）

| 术语 | 解释 |
|------|------|
| **紫微** | 智能体治理基础设施整体品牌；定位「智能体时代的安全合规基础设施」。 |
| **天枢** | 通信与身份底座：Matrix 联邦、区块链 DID、环境指纹绑定、企业 IM 审批触达。 |
| **獬豸** | 策略与审计中枢：Cedar 策略引擎、分级审批、全行为 Trace、链上存证（含私有链）。 |
| **太白** | 开放接入规范：Matrix 事件扩展、多语言 SDK、预置适配器（OpenClaw/Dify/Co-Claw 等）。 |
| **3AF** | 獬豸产品品牌名：AI Agent Audit & Firewall（AI 智能体审计与防火墙）。 |

## 二、紫微对外/对标术语（对外怎么说）

| 术语 | 全称 / 解释 | 对应紫微能力 |
|------|-------------|--------------|
| **ASPM** | Agent Security Posture Management（智能体安全态势管理） | 獬豸审计存证：监控智能体行为、发现风险、存证验真；对标 Cyera 的 DSPM。 |
| **ABP** | Agent Behavior Prevention（智能体行为防护） | 獬豸策略拦截：策略驱动的允许/拒绝/审批；对标 Cyera 的 DLP。 |
| **AIEM** | Agent Identity & Entitlement Management（智能体身份与权限管理） | 天枢身份管理：DID + 环境指纹、注册/吊销；对标 Cyera 的 CIEM。 |
| **ABAA** | Agent Behavior Analytics & Audit（智能体行为分析与审计） | 獬豸全行为 Trace + 链上存证；对标 Cyera 的 UEBA。 |

**记忆口诀**：对外讲「四 A」—— ASPM 管态势、ABP 管拦截、AIEM 管身份、ABAA 管审计；都加「Agent」前缀，对应 Cyera 的 DSPM/DLP/CIEM/UEBA。

## 三、Cyera / 数据安全行业术语（对方用啥、我们咋对应）

| 术语 | 解释 | 紫微是否做 / 怎么对应 |
|------|------|------------------------|
| **DSPM** | Data Security Posture Management，数据安全态势管理：发现、分类、评估、保护企业数据。 | 不做数据分类；紫微用 **ASPM** 管「智能体安全态势」（行为与审计）。 |
| **DLP** | Data Loss Prevention，数据泄露防护：防敏感数据未经授权流出。 | 不做内容级 DLP；紫微用 **ABP** 管「智能体行为防护」（策略拦截、审批）。 |
| **CIEM** | Cloud Infrastructure Entitlement Management，云基础设施权限管理：云 IAM 过度权限发现与收敛。 | 不做云 IAM 扫描；紫微用 **AIEM** 管「智能体身份与权限」（DID + 环境指纹）。 |
| **UEBA** | User and Entity Behavior Analytics，用户与实体行为分析：异常行为、内部威胁。 | 不做「人」的 UEBA；紫微用 **ABAA** 管「智能体行为分析与审计」（Trace + 存证）。 |
| **CSPM** | Cloud Security Posture Management，云安全态势管理：云配置合规、漏洞扫描。 | 不涉及；紫微不管云配置与漏洞。 |

**一句话**：Cyera 管「数据的安全」，紫微管「智能体的行为」；术语可类比，场景不同。

## 四、身份与链相关术语（紫微 vs 生态）

| 术语 | 解释 | 紫微对应 / 说明 |
|------|------|------------------|
| **DID** | Decentralized Identifier（W3C）：去中心化标识符，含公钥、元数据。 | 天枢为每个智能体发 DID，写私有/联盟链；含环境指纹、所有者、状态。 |
| **环境指纹** | 部署环境特征哈希（硬件/软件/网络），用于防冒充。 | 天枢将环境指纹与 DID 绑定；启动时校验，失配则身份「待验证」。 |
| **Merkle 树 / 根哈希上链** | 将一批操作指纹组成 Merkle 树，树根上链，单条记录可验真。 | 獬豸：Trace 指纹批处理 → Merkle 根上链；链上不存原始日志，仅存证。 |
| **私有链 / 联盟链** | 企业自管或少数机构共管的链，非公链。 | 紫微默认獬豸运维私有链，承载 DID 注册 + 审计存证；多机构可用联盟链。 |
| **Identity Registry** | 身份注册表（如 ERC-8004）：链上身份登记。 | 紫微：獬豸链上 DID 注册与查询，身份不可转让、非 NFT。 |
| **Reputation Registry** | 声誉注册表：链上防篡改的反馈/评分。 | 紫微当前无；可远期扩展「智能体审计/合规评分」上链。 |
| **Validation Registry** | 验证注册表：可插拔信任（如 TEE/zkML 证明）。 | 紫微用「环境指纹 + 分级审批」表达信任等级；可显式建模为验证等级。 |
| **ERC-8004** | 以太坊上 AI 代理身份与声誉的「信任层」协议（NFT 身份、声誉、验证）。 | 理念部分对齐（链上身份、可验证）；紫微不做 NFT 化、不做公链与交易。 |
| **AgentID** | 生态里常用：代理的唯一链上标识（如 ERC-721 NFT）。 | 紫微用 DID 作为智能体唯一标识，非 NFT、不交易。 |

## 五、通信与协议术语

| 术语 | 解释 | 紫微对应 / 说明 |
|------|------|------------------|
| **Matrix** | 联邦通信协议：端到端加密、跨服务器/跨组织、可扩展事件。 | 天枢基于 Matrix；紫微扩展事件：m.agent.register、identity、approval、action、audit、heartbeat、revoke。 |
| **太白协议** | 基于 Matrix 的治理事件集合（注册、身份下发、操作上报、存证回执、心跳、吊销）。 | 太白 SDK/适配器实现；智能体与天枢/獬豸交互的标准事件。 |
| **联邦** | 多域、多组织互联，无单一中心网关。 | Matrix 联邦：跨域智能体身份互信、消息路由。 |
| **A2A** | Agent-to-Agent 协议（业界趋势）。 | 紫微通过 Matrix + 太白协议实现智能体间通信与治理，与 A2A 方向一致。 |

## 六、策略与审计术语

| 术语 | 解释 | 紫微对应 / 说明 |
|------|------|------------------|
| **Cedar** | AWS 开源的 ABAC 策略语言与评估引擎。 | 獬豸核心策略引擎：主体(Principal)、动作(Action)、资源(Resource)、上下文(Context)。 |
| **ABAC** | Attribute-Based Access Control，基于属性的访问控制。 | 獬豸策略模型即 ABAC。 |
| **分级审批** | 按风险等级选择：一键通过 / 显式确认 / 生物识别+确认。 | 獬豸内置；低/中/高风险对应不同审批方式与存证要求。 |
| **全行为 Trace** | 录制智能体操作、通信、文件、命令、API 等。 | 獬豸通过钩子/适配器采集；Trace 指纹上链，原数据本地/对象存储。 |
| **操作指纹 / 存证** | 单条操作的哈希；批量 Merkle 根上链，可验真。 | 獬豸：Trace → 指纹 → 批处理 → 根上链；3 秒验真、司法级。 |

## 七、去中心化生态术语（心中有数、不混淆）

| 术语 | 解释 | 紫微是否做 / 关系 |
|------|------|-------------------|
| **ERC-8004** | 见「四、身份与链」。 | 理念对齐身份与可验证性；紫微不做 NFT 身份与公链市场。 |
| **ChaosChain SDK** | ERC-8004 实现；身份 + 支付(x402) + 意图(AP2) + 过程证明。 | 不做；紫微无代币与公链支付。 |
| **AgentGo / SIGMA 评分** | 链上智能体信誉量化与跨链身份。 | 不做；可远期做「审计/合规评分」时参考。 |
| **Recall** | 去中心化 AI 技能市场、代币质押、竞赛上链、AgentRank。 | 不做开放市场与代币；AgentRank 可作远期「能力/合规排名」参考。 |
| **AgentRank** | 基于链上表现的智能体排名与声誉协议。 | 紫微当前无；远期若做能力/合规排行可借鉴思路。 |
| **ISEK** | 六阶段协作协议：发布→发现→招募→执行→结算→反馈；NFT 身份、$ISEK。 | 不做代币与公链；阶段划分可作「能力注册/任务协作」文档描述参考。 |
| **ooder-Nexus / SuperAgent** | 边缘 P2P、轻量节点(如 128MB)、A2A、MCP。 | 紫微主形态是中心化治理+Matrix；边缘/轻量为可选扩展。 |
| **x402 / AP2** | 代理支付、意图验证协议。 | 紫微不涉及。 |
| **TEE / zkML** | 可信执行环境、零知识证明机器学习。 | 高敏感场景可要求 TEE/强校验；可与「验证等级」结合。 |
