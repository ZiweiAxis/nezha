# 太白边界决策

**版本**：v1.1  
**日期**：2026-02-13  
**状态**：已接受（v1.0 暂不独立 → v1.1 建立太白子模块）  
**关联**：ADR-004（architecture-ziwei.md）、紫微平台规划梳理 §5 动作 4

---

## 1. 决策结论（v1.1 更新）

**建立太白独立子模块** `ziwei/taibai/`，用于：

- **协议与 SDK**：实现与技术方案 §4 对齐的太白协议常量与 Python SDK 雏形（发现、注册、心跳、操作上报）。
- **接入验证**：提供**验证用智能体**（`taibai/examples/verification_agent`），配合天枢 + 谛听做**集成接入验证**（发现 → 注册/心跳 → 上报一条操作），满足常见集成验证场景。

原 v1.0「暂不独立、由天枢+文档暂担」的结论由本版替代；太白子模块与天枢、谛听并列，协议权威仍为根技术方案 §4。

---

## 2. 理由（建立子模块）

- **集成验证需求**：需要太白子模块**配合一个智能体**进行**接入验证**，单靠文档与天枢无法提供统一、可复用的验证入口。
- **职责清晰**：接入规范、SDK、验证用智能体归属太白，与天枢（通信与身份）、谛听（策略与审计）边界明确。
- **可扩展**：后续多语言 SDK、预置适配器（OpenClaw、Dify 等）可在此子模块内扩展，与技术方案 §4.4 一致。
