# 操作路径：下一步具体怎么执行

**目的**：按《下一步执行清单》给出**一步步可操作**的路径，避免「打开某目录之后不知道做什么」。

---

## 前置：Cursor 里「子 Agent」的两种用法

| 方式 | 操作 | 适用 |
|------|------|------|
| **A. 只开子项目文件夹** | 菜单 **File → Open Folder**，选 `ziwei/diting`（或 `ziwei/tianshu`、`ziwei/taibai`），用**该文件夹**作为当前工作区 | 希望所有对话、搜索、终端默认都在该子项目内 |
| **B. 保持开 ziwei 根目录** | 当前已打开 `ziwei` 根目录时，在 **Chat 里 @ 或选择对应 Agent**（diting / tianshu / taibai），并在首条消息里写明「在 diting 下执行…」 | 不想切换文件夹，靠 Agent 规则限定修改范围 |

两种方式二选一即可；下面每条路径都会写「用 A」或「用 B」时分别怎么做。

---

## 路径 1：谛听 I-009（按风险/path 配置不同超时与审批人）

### 1.1 进入「谛听上下文」

- **若用方式 A**：**File → Open Folder** → 选 `ziwei/diting`，确认左侧资源管理器根节点是 `diting`。
- **若用方式 B**：保持打开 `ziwei`，在 Chat 里 **@diting** 或从 Agent 列表选 **diting**（子 Agent）。

### 1.2 在 Chat 里输入（可直接复制）

```
请实现 ISSUE_LIST 中的 I-009：按风险等级或 path 配置不同超时与审批人。

要求：
- 不同风险等级或 path 可配置不同的超时时间、不同审批人列表。
- 与 I-008 已实现的多审批人（approval_user_ids、approval_policy）兼容，可在此基础上扩展为「按条件选用不同配置」。
- 修改范围限于 ziwei/diting：config、CHEQ、ownership/delivery 等；需要时更新 config.example.yaml 与 ISSUE_LIST。

完成后：跑 make test，通过后在 ISSUE_LIST 中将 I-009 标为 Done。
```

### 1.3 执行后你要做的

1. 看 Agent 是否只改了 `diting/` 下文件（未改 tianshu/taibai）。
2. 在**终端**（工作区为 diting 时在 diting 根下执行）：
   ```bash
   make test
   ```
3. 若通过，确认 `ISSUE_LIST.md` 里 I-009 已标 **Done**；需要时可提交并 push。

---

## 路径 2：天枢引用路径自检

### 2.1 进入「天枢上下文」

- **方式 A**：**File → Open Folder** → 选 `ziwei/tianshu`。
- **方式 B**：打开 `ziwei`，在 Chat 里 **@tianshu** 或选 **tianshu** Agent。

### 2.2 在 Chat 里输入（可直接复制）

```
请做一次引用路径自检：

1. 在 tianshu 仓库内搜索所有引用「紫微技术方案」「技术方案」「ziwei/docs」的地方（含 README、docs、design-alignment、.cursor 等）。
2. 确认这些引用都指向根技术方案：ziwei/docs/open/technical/紫微智能体治理基础设施-技术方案.md（或相对路径 docs/open/technical/...，视仓库根是 ziwei 还是 tianshu 而定）。
3. 若有错误路径、死链或过时文档名，列出并修正；若有 markdown 链接，确保可点击且指向正确文件。

输出：简要列表「文件名 + 修改内容」；若无问题则说明「未发现需修改的引用」。
```

### 2.3 执行后你要做的

- 看 Agent 给出的列表，确认修改无误（尤其是 design-alignment、README）。
- 若 tianshu 是单独打开的文件夹，相对路径可能是 `../docs/...` 或 `ziwei/docs/...`，以「从 tianshu 仓库根能正确打开该 md」为准。

---

## 路径 3：太白 P1 协议与 §4 一致

### 3.1 进入「太白上下文」

- **方式 A**：**File → Open Folder** → 选 `ziwei/taibai`。
- **方式 B**：打开 `ziwei`，在 Chat 里 **@taibai** 或选 **taibai** Agent（若已配置）。

### 3.2 在 Chat 里输入（可直接复制）

```
请做 P1 协议与文档对齐：

1. 打开根技术方案 ziwei/docs/open/technical/紫微智能体治理基础设施-技术方案.md，精读 §4（太白子系统：开放接入规范），列出所有协议约定（事件类型、SDK 能力、适配器、示例等）。
2. 在 taibai 仓库内逐项核对：sdk/python（ziwei_taibai）、examples/verification_agent、taibai/docs/ 与 §4 是否一致。
3. 偏差列出清单（协议项 + 当前实现/文档 + 建议修正）；能直接改的代码或文档在本仓库内修正；若需根仓或他仓改动，单独列出「待根/他仓」。
4. 若有 taibai/docs/P1-协议与文档对齐.md，更新之；否则在 taibai/docs 或 _bmad-output 下建一份简短对齐报告。

修改范围限于 ziwei/taibai；不修改 ziwei/docs 根技术方案。
```

### 3.3 执行后你要做的

- 看偏差清单与修正是否合理。
- 若有「待根/他仓」项，可后续在主仓或对应子项目里单独排期。

---

## 路径 4：根侧更新进度（子项目做完某事后）

你在**主仓 ziwei** 的 Chat 里（不必 @ 特定 Agent），或自己改文档即可：

1. 打开 `_bmad-output/planning-artifacts/各子项目应对焦事项与进度同步.md`。
2. 在 **§3 进度同步表** 里，把对应子项目的「当前状态」改成最新（例如谛听做完 I-009 后写「I-009 Done；下一对焦 I-010～」）。
3. 在 **§5** 对应子项目下，把已完成的项标 **Done**，并更新「上次进度同步」日期。
4. 保存；若需留档可提交。

---

## 小结表：该去哪、说什么

| 要做的事 | 在 Cursor 里「去哪」 | 在 Chat 里「说什么」 |
|----------|----------------------|------------------------|
| 谛听 I-009 | 开文件夹 `ziwei/diting` 或 @diting | 复制上面「路径 1」的输入框内容 |
| 天枢引用自检 | 开文件夹 `ziwei/tianshu` 或 @tianshu | 复制上面「路径 2」的输入框内容 |
| 太白 P1 协议对齐 | 开文件夹 `ziwei/taibai` 或 @taibai | 复制上面「路径 3」的输入框内容 |
| 根侧更新进度 | 保持 ziwei 根目录，改《各子项目应对焦事项与进度同步》 | 按路径 4 手动编辑 §3/§5 |

按上述「去哪 + 说什么」执行即可一步步推进。

**若当前 Cursor 里没有 @Agent 或子 Agent 选项**：可用方式 A（File → Open Folder 只开 `ziwei/diting` 或 `ziwei/tianshu` 等），在同一 Chat 里直接粘贴上面对应路径的「输入框内容」；Agent 会根据当前打开的工作区识别是 diting/tianshu/taibai，并只改该目录下文件。
