# 子项目任务消费快速参考（运行态）

**目的**：运行中的天枢向 Agent 投递指令/审批时，Agent 如何接收（平台当前能力）。

---

## 1. 当前消费方式

| 方式 | 说明 |
|------|------|
| **Matrix Room 收消息** | 天枢通过 `send_agent_message` 发到该 Agent 的 Matrix 房间；太白/悟空等接入该 Room 收事件即可。 |
| **审批请求** | 獬豸调 `POST /api/v1/delivery/approval-request`，天枢投递到 DELIVERY_ROOM_ID；用户/子项目在对应 Matrix 房间内看到。 |

仅通过 **Matrix 收消息**及上述投递接口接收。

---

## 2. 投递消息的 payload 语义

| type | 说明 | payload 常见字段 |
|------|------|------------------|
| **approval_request** | 审批请求（如獬豸 CHEQ） | request_id, title, description, approve_url, reject_url |
| **instruction** | 执行指令 | body, title |
| **notification** | 通知 | body, title |

解析 Matrix 事件时按 msgtype 或 payload 内 type 处理。
