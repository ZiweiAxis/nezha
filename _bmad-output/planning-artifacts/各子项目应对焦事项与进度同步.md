## 4. 与其它文档的关系

- **本文**：限制对焦「各子项目下面应该做什么」+ **进度是否同步过** 的单一入口；保持简短，只列本阶段应对焦项。
- **《下一步执行清单》**：根侧维护的「当前可执行任务」列表，与本文 §3/§5 一致，供子 Agent 与根侧按单推进。
- **《根与子模块规划对齐》**：引用一致、冲突与遗漏、引用路径；与本文互补，不重复列详细 Issue。
- **《platform-epics-and-submodule-mapping》**：平台 Epic 与子模块 Issue 映射；本文的「应对焦事项」来自该映射的当前阶段拆解。
- **子项目 ISSUE_LIST / pm-what-to-do-next**：进度真实来源；根侧「进度同步」即把上述子项目状态汇总到本文 §3。
